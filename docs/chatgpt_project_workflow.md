# ChatGPT Project Workflow

This workflow bridges the local AI Newspaper batch pipeline with ChatGPT Project
analysis without adding paid API requirements to the application runtime.

## Rules

- The ChatGPT Project chat must include the GitHub repository
  `Oni508/ai-newspaper`.
- Do not use Deep Research.
- Use only the files generated by GitHub Actions:
  - `prompt.md`
  - `news_payload.json`
  - `schema.json`
- ChatGPT output must be JSON only.
- The JSON must follow `schemas/chatgpt_analysis.schema.json`.
- Do not output stock price forecasts, buy recommendations, sell
  recommendations, hold recommendations, target prices, or trading timing.

## Prepare Briefing Inputs

The `Prepare ChatGPT briefing` workflow runs the fetch step and builds a
briefing package.

It creates:

```text
artifacts/briefing_inputs/{edition_id}/prompt.md
artifacts/briefing_inputs/{edition_id}/news_payload.json
artifacts/briefing_inputs/{edition_id}/schema.json
artifacts/briefing_inputs/{edition_id}/issue_body.md
```

The workflow uploads the package as a short-lived artifact and creates an Issue
whose title starts with:

```text
[AI Newspaper Input]
```

## ChatGPT Project Step

1. Open a ChatGPT Project chat that includes `Oni508/ai-newspaper`.
2. Do not enable Deep Research.
3. Paste the GitHub Actions generated `prompt.md`.
4. Use the generated `news_payload.json` and `schema.json` as the only analysis
   inputs.
5. Ask ChatGPT to return JSON only.
6. Confirm the JSON follows `schema.json`.

## Submit Analysis

Paste the ChatGPT result into a comment on the generated Issue.

Use this exact marker:

````markdown
AI_NEWSPAPER_ANALYSIS_JSON

```json
{
  "edition_id": "example",
  "topics": []
}
```
````

The actual JSON must be the full ChatGPT output and must conform to
`schemas/chatgpt_analysis.schema.json`.

## Render Result

The `Render ChatGPT analysis` workflow starts from an `issue_comment` event only
when all of these are true:

- The Issue title starts with `[AI Newspaper Input]`.
- The comment body contains `AI_NEWSPAPER_ANALYSIS_JSON`.
- The comment author is the repository owner.

The workflow extracts the fenced JSON block, validates it against
`schemas/chatgpt_analysis.schema.json`, extracts the original `news_payload.json`
from the Issue body, writes `analysis_output.json`, renders an HTML digest with
real source URLs resolved from `source_refs`, uploads the files as an artifact,
and posts a comment with the workflow run link.

The rendered artifact includes:

```text
analysis_output.json
news_payload.json
digest.html
```

Artifacts are retained for 2 days.

## Local Verification

Run:

```bash
uv run pytest
uv run ai-newspaper fetch
python scripts/build_briefing_package.py
```

Confirm these files exist under `artifacts/briefing_inputs/.../`:

```text
prompt.md
news_payload.json
schema.json
issue_body.md
```

Also confirm workflow YAML files are syntactically valid before pushing.
